# Project Documentation: AI-Enhanced Interview Management System

## 1. Problem Statement

Recruitment is a critical business process for any organization. Traditional recruitment processes involve high administrative overhead, including manual interview scheduling, lack of centralized candidate tracking, and difficulty in aligning interviewers, recruiters, and candidates. Furthermore, hiring managers often struggle to systematically evaluate candidate attributes over multiple rounds.

This project addresses these challenges by developing a centralized, web-based **AI-Enhanced Interview Management System** designed to:
- Schedule, track, and manage interview rounds seamlessly.
- Apply **Machine Learning** (Random Forest Classifier) to forecast candidate selection likelihoods based on attributes like years of experience, technical assessment scores, communication, and education.
- Utilize **Generative AI** (LLM API) to automatically generate candidate feedback summaries and HR evaluation reports, saving interviewers and HR managers significant documentation time.

---

## 2. System Architecture

The system is designed with a decoupled frontend-backend architecture:

1. **Client Layer (Frontend)**:
   - Dynamic Vanilla HTML5 structure.
   - Sleek styling with custom HSL variables, dark-mode styling, glassmorphism, responsive grid layouts, and hover micro-animations.
   - Pages: Landing Choice (`index.html`), Candidate Portal (`candidate.html`), Recruiter/Interviewer Panel (`recruiter.html`).
   - AJAX communications through Vanilla JavaScript (`api.js`, `landing.js`, `candidate.js`, `recruiter.js`).

2. **Server Layer (Backend)**:
   - **FastAPI Framework**: High performance, modern API server with autogenerated Swagger UI.
   - **SQLAlchemy ORM**: Interfaces with SQLite to perform CRUD operations on Users, Candidate Profiles, Scheduled Interviews, and Feedback records.
   - **ML Prediction Module**: Loads the trained Random Forest Classifier, processes incoming candidate metrics, and maps outcomes.
   - **GenAI Report Engine**: Interfaces with the Gemini API to construct HR summaries, or triggers a local natural language generator fallback.

3. **Data & ML Layer**:
   - **SQLite Database**: Stored in `Backend/interviews.db` for state persistence.
   - **Dataset (`interview_dataset.csv`)**: 1,000 synthetic recruitment profiles containing experience, skills matching, technical scores, communication ratings, previous ratings, education, and target selection probability.
   - **Model Serialization**: Pickled model and scaler objects (`ML/candidate_selection_model.pkl`).

## 3. Machine Learning Methodology

### Algorithm Choice
The **Random Forest Classifier** was selected due to its robustness to noise, resilience to overfitting, and ability to handle tabular categorical and numerical attributes effectively without requiring complex scaling.

### Features Utilized
1. `years_of_experience` (Numerical: 0-15 years)
2. `skills_score` (Numerical: 30-100 match percentage)
3. `technical_score` (Numerical: 40-100 evaluation score)
4. `communication_rating` (Categorical/Ordinal: 1-5 rating)
5. `previous_performance` (Categorical/Ordinal: 1-5 rating)
6. `education_level_encoded` (Categorical/Ordinal: Bachelors=0, Masters=1, PhD=2)

### Target Variables
- `0`: Low Selection Probability
- `1`: Moderately Likely to Select
- `2`: Highly Likely to Select

### Model Training & Evaluation Metrics
- **Dataset Size**: 1000 records
- **Split Ratio**: 80% Train, 20% Test (Stratified)
- **Model Parameters**: 100 decision trees, maximum depth of 8
- **Model Accuracy**: **78.50%**
- **Evaluation Details**:
  - *Low Selection Probability*: Precision = 89%, Recall = 65%
  - *Moderately Likely to Select*: Precision = 74%, Recall = 96%
  - *Highly Likely to Select*: Precision = 100%, Recall = 29%
  - *Model Pickle Path*: `ML/candidate_selection_model.pkl`

## 4. Generative AI Module

### Functionality
After an interviewer evaluates a candidate and submits their scores, the GenAI Module generates:
1. **Candidate Interview Summary**: Paragraph summarizing performance and suitability.
2. **Key Strengths**: Specific bullets highlighting candidate qualifications.
3. **Improvement Areas**: Construction of guidance for the candidate.
4. **Final Observations Report**: Full printable narrative evaluation.

### Prompt Engineering
The module uses structured prompts containing:
- Candidate name, role, skills, experience, and educational level.
- Numerical evaluation scores (technical, communication, previous performance).
- Interviewer notes.
- Machine Learning selection probability class.

It prompts the model to return a strict JSON payload with appropriate keys.

### Resilience & Offline Fallback
To ensure the system works reliably out-of-the-box in offline environments or without an active `GEMINI_API_KEY`, we implemented a **hybrid generator** in `Backend/genai_report.py`. If the API key is not configured or the connection times out, a local natural language template engine runs, producing a customized, detailed summary report utilizing the candidate's metrics and interviewer comments.
